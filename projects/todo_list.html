Projeto Todo-List

Criar uma aplicação para organizar tarefas parece ser um marco na vida de qualquer desenvolvedor. Não foi diferente no meu caso. Nesse artigo você vai ver os desafios enfrentados nesse projeto, e como ele contribuiu para o meu crescimento.

O projeto Todo-List realmente foi um projeto desafiador e que contribuiu muito para o meu aprendizado. É claro que não há nada mais prazeroso pra um desenvolvedor do que criar uma aplicação que funcione bem e que seja algo útil. Com nossas vidas aterefadas um organizador de tarefas é sempre útil. Então, com certeza esse foi um projeto divisor de águas para mim.

O desenvolvimento consistia na aplicação de várias técnicas aprendidas até então, só que dessa vez em conjunto. Dentre os conhecimentos aprendidos que foram colocados em prática, alguns deles são:

  • HTML semântico,
  • Seletores CSS e posicionamento de elementos,
  • Métodos de seleção de elementos do DOM,
  • Escuta de eventos e manipulação de elementos com JavaScript,
  • Modificação de atributos e classes dos elementos HTML, e
  • Utilização do Web Storage.

Com certeza foi ótimo aplicar todas esses métodos ao mesmo tempo. Serviu e muito para gravar bem como essas técnicas funcionam e quando utilizar cada uma delas. Mas nesse ponto pra mim uma novidade foi a utilização de Web Storage. Neste caso o objeto localStorage foi utilizado para salvar os items da lista de tarefas do usuário no seu próprio navegador.

Dentre os requisitos do projeto estavam a manipulação dos items da lista, o que incluía seleção, movimentação, adição e exclusão de itens. Uma das funcionalidades presentes também estava a possibilidade de apenas marcar os itens como concluídos em vez de excluí-los.

Você poderá ver o código desenvolvido neste repositório do GitHub. O arquivo README.md explica com detalhes os requisitos obrigatórios e bônus do projeto proposto. Sinta-se à vontade para revisar e comentar o código - críticas construtivas são sempre bem-vindas. E lembre-se de que esse projeto foi desenvolvido no início da minha jornada para me tornar um desenvolvedor Web.  Como venho falado nos meus artigos, o objetivo é mostrar a minha trajetória como desenvolvedor, e o meu crescimento ao longo do tempo.

Conclusão

O desenvolvimento dessa página foi algo que realmente me trouxe muita realização. Embora seja um projeto simples e curto, pela primeira vez eu senti que já poderia desenvolver algo útil, bonito e funcional com o conhecimento adquirido até então. Com certeza esse projeto me deu uma base muito boa para os conhecimentos mais avançados que ainda viriam.
